# 更改计划

## 需求描述
扫描音频文件时，若艺术家（Artist）尚未设定图片路径（CoverPath），则自动将其设定为该艺术家下第一个有效歌曲的专辑图片路径。确保每个艺术家至少有一张代表性的封面图片。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/Models/Artist.cs` — 艺术家模型，包含 CoverPath 属性
  - `seamless loop music/Models/Album.cs` — 专辑模型，包含 CoverPath 属性
  - `seamless loop music/Data/DatabaseHelper.cs` — 数据库操作，包含 `UpsertArtistAlbum` 方法（第150-167行），负责 Artist 和 Album 的插入/更新
  - `seamless loop music/Services/TrackMetadataService.cs` — 元数据服务，负责提取曲目封面（`GetOrExtractCover` 方法）

## 当前项目状态
- **现有实现描述**：
  - `Artist` 模型已定义 `CoverPath` 属性（`Artist.cs:9`）
  - `Album` 模型已定义 `CoverPath` 属性（`Album.cs:10`）
  - 在 `DatabaseHelper.UpsertArtistAlbum` 方法中（第150-167行）：
    - 第155行：仅插入 Artist 的 Name，未设置 CoverPath
    - 第158-164行：插入/更新 Album 时设置了 CoverPath，并有逻辑避免覆盖已有值
  - `TrackMetadataService.GetOrExtractCover` 方法负责提取曲目封面，并缓存到本地或读取已有缓存
  - 数据库查询 `FullTrackSelect`（第103-115行）中，Artist 信息仅用于显示，未使用其 CoverPath

- **可复用的逻辑**：
  - `UpsertArtistAlbum` 方法中的 Artist 插入逻辑可扩展，增加 CoverPath 更新
  - `GetOrExtractCover` 方法已有完善的封面提取和验证逻辑（检查文件有效性）
  - Album 的 CoverPath 更新逻辑（`ON CONFLICT...DO UPDATE SET CoverPath = COALESCE(Albums.CoverPath, excluded.CoverPath)...`）可作为参考

## 已确认方案

**✅ 采用方案A**：修改 `DatabaseHelper.UpsertArtistAlbum` 方法

**确认要点**：
- Artist 的 CoverPath 使用传入的 `coverPath` 参数（即 `track.CoverPath`）
- 在当前代码中，`track.CoverPath` 通过 `TrackMetadataService.GetOrExtractCover` 提取，对于同专辑的曲目会共享同一个缓存文件
- 因此 `track.CoverPath` 和 `Album.CoverPath` 在逻辑上是一致的，都是该专辑的封面路径
- 如果 Artist 下有多个 Album，**始终使用第一个扫描到的 Album 的封面**（通过 `WHERE CoverPath IS NULL OR CoverPath = ''` 实现）

## 实现计划
1. 修改 `DatabaseHelper.UpsertArtistAlbum` 方法（第150-167行）
2. 第155行：修改 Artist 插入语句，增加 CoverPath 字段：
   ```csharp
   // 修改前
   db.Execute("INSERT OR IGNORE INTO Artists (Name) VALUES (@Name)", ...);
   
   // 修改后
   db.Execute("INSERT OR IGNORE INTO Artists (Name, CoverPath) VALUES (@Name, NULL)", new { Name = artistName }, transaction: trans);
   ```
3. 在获取 `artistId` 后（第156行后），添加 Artist CoverPath 更新逻辑：
   ```csharp
   // 若传入的 coverPath 有效，且 Artist 当前无封面，则更新
   if (!string.IsNullOrEmpty(coverPath))
   {
       db.Execute(@"UPDATE Artists 
                   SET CoverPath = @Cover 
                   WHERE Name = @Name AND (CoverPath IS NULL OR CoverPath = '')", 
                   new { Name = artistName, Cover = coverPath }, transaction: trans);
   }
   ```
4. 确认 `SaveTrack`（第130-148行）和 `BulkInsert`（第171-196行）方法中调用 `UpsertArtistAlbum` 时传入的 `coverPath` 参数已经是有效的专辑封面路径（当前已传入 `track.CoverPath`）
5. 验证：运行扫描，检查数据库 Artists 表的 CoverPath 是否被正确设置

## 预期结果
- 扫描音频文件后，数据库 Artists 表的 CoverPath 字段会被自动填充为第一个有效歌曲的专辑图片路径
- 在 UI 显示艺术家时，可以正确加载并显示艺术家的封面图片
- 对于尚未扫描到封面的艺术家，CoverPath 保持为 NULL 或空字符串，等待后续扫描时更新

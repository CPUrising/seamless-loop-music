# 更改计划

## 需求描述
完善系统托盘（NotifyIcon）功能，包括：最小化窗口到托盘而非任务栏、明确关闭窗口时的行为（直接退出 vs 最小化到托盘）、优化托盘右键菜单，并可能添加相关设置项供 CPU 大人选择行为模式。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/Services/NotifyIconService.cs` — 当前托盘服务实现
  - `seamless loop music/Services/INotifyIconService.cs` — 托盘服务接口
  - `seamless loop music/UI/MainWindow.xaml.cs` — 主窗口关闭/最小化逻辑
  - `seamless loop music/App.xaml.cs` — 应用生命周期与托盘初始化
  - `seamless loop music/Services/AppStateService.cs` — 应用状态持久化（可保存托盘相关设置）
  - `seamless loop music/UI/Views/SettingsWindow.xaml.cs` — 设置窗口（可添加托盘行为设置项）

## 当前项目状态
- 现有实现描述：
  - 托盘服务 `NotifyIconService` 已实现基本功能：显示托盘图标、左键弹出 `TrayControlsWindow`、双击恢复主窗口、右键菜单包含播放/暂停/上一首/下一首/退出等基础控制项。
  - 主窗口 `MainWindow` 在 `Closing` 事件中仅保存应用状态，未处理最小化到托盘的逻辑。
  - 托盘右键菜单目前功能较简单，缺少"显示主窗口"、"最小化到托盘"等常用选项。
  - 应用关闭时（`Exit` 菜单项）直接调用 `Application.Current.Shutdown()`，无"最小化到托盘"的选项。
  - 目前没有设置项来控制关闭窗口时的行为。
- 可复用的逻辑：
  - `NotifyIconService` 中已有的 `NotifyIcon_DoubleClick` 方法可用于恢复主窗口，可复用为"显示主窗口"菜单项逻辑。
  - `AppStateService` 的 `SaveCurrentStateAsync` / `RestoreStateAsync` 方法模式可用于保存/恢复托盘行为设置（如"关闭时最小化到托盘"开关）。
  - `SettingsWindow` 已有的语言设置绑定模式可用于快速添加新设置项。

## 可选实现方案
- 方案A：添加"关闭时最小化到托盘"设置项，默认行为为直接退出
  - 描述：在设置页面添加开关 `CloseToTray`，关闭主窗口时根据此设置决定是退出还是最小化到托盘。
  - 优点：保留当前默认行为（关闭=退出），同时给予 CPU 大人选择权，符合用户习惯的多样性。
  - 缺点：需要添加设置项 UI 和状态持久化逻辑，稍微增加复杂度。

- 方案B：将关闭按钮行为改为始终最小化到托盘，只能通过托盘菜单退出
  - 描述：拦截主窗口 `Closing` 事件，取消关闭并隐藏窗口到托盘；移除托盘菜单中的"Exit"或保留为唯一退出方式。
  - 优点：实现简单，无需额外设置项。
  - 缺点：可能让 CPU 大人困惑，不知道如何真正退出应用；违背 Windows 应用的常规交互习惯。

- 方案C：智能判断，播放时最小化到托盘，未播放时直接退出
  - 描述：关闭窗口时检查 `PlaybackState`，若正在播放则最小化到托盘，否则直接退出。
  - 优点：智能体验，减少误操作。
  - 缺点：逻辑不够明确，CPU 大人可能无法预期关闭行为；调试和维护成本较高。

## 实现计划
1. **扩展接口**：在 `INotifyIconService` 中添加 `ShowMainWindow()` 和 `HideMainWindow()` 方法定义，用于托盘菜单控制主窗口显示/隐藏。

2. **实现窗口控制方法**：在 `NotifyIconService` 中实现上述方法：
   - `ShowMainWindow()`：将主窗口状态恢复为 Normal，调用 `Activate()` 和 `Show()`
   - `HideMainWindow()`：隐藏主窗口（`MainWindow.Hide()`），可选从任务栏移除（`ShowInTaskbar = false`）

3. **修改托盘右键菜单**：
   - 添加"显示主窗口"/"隐藏到托盘"动态菜单项（根据窗口可见性切换文字）
   - 保留现有播放控制菜单项（播放/暂停/上一首/下一首）
   - 保留"退出"菜单项（始终完全退出应用）

4. **添加两个独立设置项**（在 `AppStateService` 中）：
   - `MinimizeToTray`：最小化窗口时是否到托盘（默认：false）
   - `CloseToTray`：关闭窗口时是否到托盘（默认：false，即直接退出）

5. **修改 `MainWindow` 行为**：
   - 监听 `StateChanged` 事件：若窗口最小化且 `MinimizeToTray=true`，则隐藏窗口并取消最小化状态
   - 监听 `Closing` 事件：若 `CloseToTray=true` 且窗口可见，则取消关闭、隐藏窗口到托盘；否则执行原有退出逻辑（先保存状态）

6. **在 `SettingsWindow` 中添加设置项**：
   - 添加两个 `ToggleSwitch` 控件（MinimizeToTray / CloseToTray）
   - 使用与语言设置相同的绑定模式，保存到 `AppStateService`
   - 设置变更时调用 `SaveCurrentStateAsync()` 持久化

7. **持久化逻辑**：在 `AppStateService` 中：
   - `SaveCurrentStateAsync()` 添加保存 `MinimizeToTray` 和 `CloseToTray` 设置
   - `RestoreStateAsync()` 添加恢复这两个设置
   - 使用 `IDatabaseHelper.SetSetting("App.MinimizeToTray", value)` 和 `GetSetting()` 方法

8. **确保退出清理**：在 `NotifyIconService` 的 `Exit` 菜单项和 `Application.Current.Shutdown()` 前，调用 `Dispose()` 释放托盘图标资源。

## 已确认需求
- ✅ 选择方案A：添加设置项，默认行为为直接退出
- ✅ 关闭主窗口默认行为：直接退出（CloseToTray 默认 false）
- ✅ 两个独立设置项：MinimizeToTray 和 CloseToTray 分开控制
- ✅ 托盘图标 ToolTip：固定为 "Seamless Loop Music"，不显示播放信息
- ✅ 不需要：托盘图标控制音量或查看播放进度

## 预期结果
- CPU 大人可通过设置页自由配置两个独立行为：
  - **最小化到托盘**：点「_」按钮时藏到托盘（默认关）
  - **关闭到托盘**：点「X」按钮时藏到托盘（默认关，即直接退出）
- 托盘右键菜单包含"显示/隐藏主窗口"选项，可快速切换窗口可见性
- 相关设置能够持久化保存，重启应用后依然生效
- 若开启对应设置，窗口将从任务栏消失，仅保留托盘图标，点击托盘或菜单可恢复
- 无论设置如何，"退出"菜单项始终完全关闭应用并清理资源

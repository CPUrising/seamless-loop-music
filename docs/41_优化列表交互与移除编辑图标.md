## 可选实现方案

- 方案A：代码后置文件处理事件（简单直接）
  - 在 `TrackListView.xaml.cs` 中处理 `MouseDoubleClick` 和封面点击事件
  - 通过 `DataContext` 调用 ViewModel 方法
  - 优点：实现简单，符合 WPF 惯例，无需额外依赖
  - 缺点：代码后置文件包含逻辑，不完全符合 MVVM 纯粹性

- 方案B：使用 Interaction Behavior（MVVM 推荐）
  - 引入 `Microsoft.Xaml.Behaviors.Wpf` 包（如未引用）
  - 使用 `EventTrigger` + `InvokeCommandAction` 绑定事件到 ViewModel 命令
  - 优点：完全符合 MVVM 模式，ViewModel 逻辑清晰
  - 缺点：需要引入新的依赖包，实现稍复杂

- 方案C：在 ItemContainerStyle 中使用 EventSetter
  - 在 `ListBoxItem` 样式中添加 `MouseDoubleClick` 事件设置
  - 优点：纯 XAML 实现
  - 缺点：EventSetter 不支持绑定到 ViewModel 命令，仍需代码后置

**推荐方案A**：项目已有代码后置文件 `TrackListView.xaml.cs`，且 `MultiSelectListBox` 可能已处理多选逻辑，使用代码后置最简单直接。

1. **删除编辑图标**：
   - 在 `TrackListView.xaml` 中删除第112-116行的编辑按钮（IconEdit 按钮）

2. **添加封面图片到 DataTemplate**：

4. **实现点击封面进入 NowPlaying**：
   - 在封面图片的点击事件中，通过 `RegionManager` 导航到 `NowPlayingView`

5. **调整单击逻辑**：
   - 确保单击仅选中，不触发播放或导航
   - 当前 `SelectedTrack` setter 中的 `OnOpenDetail` 调用仅在 `IsCompact=true` 时执行，正常模式无需修改

6. **添加导航命令到 ViewModel**：
   - 在 `TrackListViewModel.cs` 中添加 `GoToNowPlayingCommand`
   - 实现导航到 NowPlayingView 的逻辑

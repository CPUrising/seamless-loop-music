# 更改计划

## 需求描述
规范项目的颜色管理方式，参考 Dopamine 项目的最佳实践，采用分层资源架构、两阶段定义法和命名规范来管理颜色资源，消除硬编码，支持更好的主题管理。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/UI/Themes/Colors.xaml` — 当前颜色与画刷定义
  - `seamless loop music/UI/Themes/Styles.xaml` — 样式定义，引用画刷资源
  - `seamless loop music/UI/Themes/Controls.xaml` — 控件模板资源
  - `seamless loop music/App.xaml` — 应用资源合并配置
- 参考项目的源文件：
  - Dopamine 项目的 `Common.xaml` — 基础颜色定义
  - Dopamine 项目的 `Light.xaml` / `Dark.xaml` — 主题调色板

## 当前项目状态
- 相关功能的现有代码情况：
  - 已采用两阶段定义法：`Colors.xaml` 中先定义 `Color_` 资源，再定义 `Brush_` 画刷
  - 命名规范正确：使用 `Color_` 前缀定义颜色，`Brush_` 前缀按用途命名画刷
  - 存在两处硬编码画刷（`Colors.xaml:30-31`）：
    ```xaml
    <SolidColorBrush x:Key="BrushBorder" Color="#313244"/>
    <SolidColorBrush x:Key="BrushSeparator" Color="#45475A"/>
    ```
  - 画刷引用使用 `StaticResource` 而非 `DynamicResource`（`Colors.xaml:18-28`），不支持运行时主题切换
  - 缺少主题分层：只有一个 `Colors.xaml`，未像 Dopamine 那样分 `Light.xaml` / `Dark.xaml`
  - `App.xaml` 中已合并 MaterialDesign 主题和自定义主题资源，但自定义资源使用 `StaticResource` 引用

## 已选定方案
**方案B：完整分层架构** — 支持运行时明暗主题切换

## 实现计划（方案B）

### 阶段一：创建基础架构
1. 创建 `UI/Themes/Common.xaml`：
   - 定义基础颜色（白、黑、灰等中性色）
   - 使用 `Color_` 前缀命名
   - 这些颜色在各个主题中保持不变

2. 完善 `UI/Themes/Colors.xaml`（Dark 主题）：
   - 使用 `MergedDictionaries` 合并 `Common.xaml`
   - 定义暗色主题专属颜色（如 `ColorBackground`、`ColorSurface` 等）
   - 所有画刷使用 `DynamicResource` 引用颜色（而非 `StaticResource`）
   - 补充缺失的 `Color_Border` 和 `Color_Separator` 定义
   - 修改 `BrushBorder` 和 `BrushSeparator` 引用对应的 Color 资源

3. 创建 `UI/Themes/Light.xaml`（粗略设定）：
   - 定义亮色主题专属颜色
   - 保持与 Dark 主题相同的 Brush 键名
   - 初期可简单复制 Dark.xaml 并调整颜色值

### 阶段二：主题切换支持
4. 修改 `App.xaml` 资源合并配置：
   - 移除直接引用 `Colors.xaml`
   - 改为根据当前主题动态加载 `Dark.xaml` 或 `Light.xaml`
   - 或通过代码动态切换资源字典

5. 创建主题管理服务（可选但推荐）：
   - 实现 `IThemeService` 接口
   - 提供 `SwitchTheme(ThemeType)` 方法
   - 保存用户主题偏好到配置

### 阶段三：验证与测试
6. 验证 `Styles.xaml` 和 `Controls.xaml` 中的引用：
   - 确保所有画刷引用使用 `DynamicResource`
   - 检查是否有硬编码的颜色值

7. 测试：
   - 应用启动和资源加载正常
   - 运行时切换主题功能正常
   - 明暗主题下 UI 显示正确

## 待确认问题
- 主题切换的触发方式：通过设置页面切换，还是增加快捷键/按钮？
- `Light.xaml` 的颜色值是否需要参考某个现有亮色主题（如 Material Design 默认亮色）？
- 是否需要为强调色（Accent）也支持主题切换，还是保持固定强调色？

## 预期结果
- 实现分层资源架构：
  - `Common.xaml` — 基础颜色（白、黑、灰）
  - `Colors.xaml` (Dark.xaml) — 暗色主题调色板
  - `Light.xaml` — 亮色主题调色板（粗略设定）
- 所有颜色都有对应的 `Color_` 定义，彻底消除硬编码
- 画刷使用 `DynamicResource` 引用颜色，支持运行时主题切换
- 明暗主题使用相同的 Brush 键名，只需切换资源字典即可整体换色
- XAML 中只需引用 `Brush_` 键名，无需关心具体颜色值
- 用户可在运行时自由切换明暗主题 (๑•̀ㅂ•́)و✧

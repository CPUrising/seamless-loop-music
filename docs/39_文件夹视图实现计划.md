# 更改计划

## 需求描述
在播放器最左侧导航栏增加"文件夹"视图，允许用户通过文件夹层级结构浏览和播放音乐。用户可以在左侧选择根文件夹，查看子文件夹树，并在右侧查看和播放该文件夹下的所有音乐文件。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/UI/Views/LibraryView.xaml` - 主界面三栏布局
  - `seamless loop music/UI/ViewModels/LibraryViewModel.cs` - 导航逻辑
  - `seamless loop music/UI/ViewModels/TrackListViewModel.cs` - 曲目列表逻辑
  - `seamless loop music/Models/CategoryItem.cs` - 分类项模型
  - `seamless loop music/Models/CategoryNavTarget.cs` - 导航目标模型
  - `seamless loop music/Data/DatabaseHelper.cs` - 数据库操作（已有MusicFolders表）
  - `seamless loop music/UI/FolderManagerWindow.xaml.cs` - 现有文件夹管理窗口

- 参考项目的源文件：
  - `CollectionFolders.xaml` - 左右分栏布局（MultiPanePanel）
  - `CollectionFoldersViewModel.cs` - 继承TracksViewModelBase
  - `IFoldersService.cs` / `IFileService.cs` - 服务层接口

## 当前项目状态
- **现有实现**：
  - LibraryView采用三栏布局：导航栏(65px) + 分类项(280px) + 曲目列表(*)
  - 导航栏支持Album/Artist/Playlist三种分类类型
  - 数据库已有MusicFolders表存储用户收藏的根文件夹
  - FolderManagerWindow可用于管理播放列表关联的文件夹
  - TrackListViewModel处理曲目列表的显示、搜索、播放等操作

- **可复用的逻辑**：
  - `CategoryNavTarget` 和 `CategoryItem` 模型结构可直接扩展
  - `LibraryViewModel` 的导航切换逻辑可复用
  - `TrackListViewModel` 的曲目显示、搜索、播放逻辑可复用
  - 数据库 `MusicFolders` 表可直接使用，无需新建表
  - `CategoryItemSelectedEvent` 事件机制可用于通信

## 实现方案

**采用方案：作为新的Prism区域视图**

- 创建独立的 `FolderTrackListView.xaml` 和 `FolderTrackListViewModel.cs`
- 通过Prism区域导航系统加载到右侧TrackList区域
- 该视图内部实现左右分栏布局（左：子文件夹列表，右：歌曲列表）
- 与Album/Artist/Playlist视图平级，职责清晰，易于维护

## 预期结果
1. 用户点击左侧导航栏的"文件夹"图标后，**中间栏显示所有收藏的根文件夹列表**（如 D:\Music、E:\Audio 等）
2. **选择根文件夹**：点击某个根文件夹后，右侧区域才展示该文件夹的内容
3. 右侧区域分为左右两栏：
   - 左栏：该根文件夹下的**子文件夹列表**（只显示当前层级，不递归），顶部有"返回上级"按钮
   - 右栏：当前文件夹下的所有音频文件列表（支持排序、搜索）
4. 点击子文件夹切换到该文件夹，点击"返回上级"回到父目录
5. **播放逻辑**：双击播放某首歌时，播放队列自动切换为该文件夹下的所有歌曲（按顺序播放）
6. 播放状态实时同步，正在播放的歌曲和所在文件夹高亮显示
7. 关闭应用后再打开，自动恢复到上次浏览的文件夹位置（包括选中的根文件夹和当前子文件夹路径）
8. 整体交互风格与现有的Album/Artist/Playlist视图保持一致 (๑•̀ㅂ•́)و✧

# 更改计划

## 需求描述
为项目中的所有按钮增加完整的三态交互反馈效果：鼠标悬停（Hover）高亮反馈、鼠标按下（Pressed）明显反馈、鼠标松开（Released）恢复反馈。当前部分按钮样式缺少按下状态的视觉反馈，导致点击体验不一致。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/UI/Themes/Colors.xaml` - 颜色资源定义
  - `seamless loop music/UI/Themes/Styles.xaml` - 按钮样式（PrimaryButton、SecondaryButton）
  - `seamless loop music/UI/Themes/Controls.xaml` - 控件样式（IconButton、GhostIconButton、ListBoxItem）
  - `seamless loop music/App.xaml` - 主题配置（MaterialDesign BundledTheme + MaterialDesign2.Defaults）
  - `seamless loop music/UI/Controls/PlaybackControls.xaml` - 播放控制按钮
  - `seamless loop music/UI/Controls/VolumeControls.xaml` - 音量控制
  - `seamless loop music/UI/Controls/TrackInfoControl.xaml` - 曲目信息控件
  - `seamless loop music/UI/Controls/ProgressControls.xaml` - 进度条控件

- 参考项目的源文件：无特定参考项目，基于 Material Design 5.3.1 的 Ripple 效果作为参考

## 当前项目状态
- 相关功能的现有代码情况：
  1. **IconButton**（`Controls.xaml:5-30`）：已有 `IsMouseOver`（背景变 BrushSurfaceHighlight，前景变 BrushAccent）和 `IsPressed`（Opacity=0.8）触发器，但按下反馈较微弱
  2. **GhostIconButton**（`Controls.xaml:33-36`）：继承 IconButton，背景透明，前景 BrushTextSecondary，按下效果相同
  3. **PrimaryButton**（`Styles.xaml:16-41`）：仅有 `IsMouseOver`（背景变 BrushAccentHover）和 `IsEnabled`，**缺少 IsPressed 触发器**
  4. **SecondaryButton**（`Styles.xaml:43-64`）：仅有 `IsMouseOver`（背景变 BrushSurfaceHighlight），**缺少 IsPressed 触发器**
  5. **ModernListBoxItem**（`Controls.xaml:39-62`）：仅有 `IsMouseOver` 和 `IsSelected`，**缺少 IsPressed 反馈**
  6. **SidebarListBoxItem**（`Controls.xaml:65-109`）：有选中/悬停效果，但**缺少按下反馈**

- 可复用的逻辑：
  - 现有的颜色资源（BrushAccent、BrushAccentHover、BrushSurface、BrushSurfaceHighlight）可以直接使用
  - `ColorSelection (#2D3F5E)` 可以作为按下状态的颜色
  - Material Design 的 Ripple 效果可以通过 `materialDesign:ButtonAssist.UniformCornerRadius` 等附加属性启用

## 选定方案

**方案B：使用 Material Design 原生的 Button 样式和 Ripple 效果**

- 描述：将项目中的 Button 替换为 Material Design 的 `Button` 样式（带 Ripple 效果），使用 `materialDesign:ButtonAssist` 附加属性来实现透明度降低的按下效果
- 优点：获得原生的 Material Design Ripple 点击效果，视觉体验更好，与主题更统一
- 按下效果：透明度降低（CPU 大人偏好）
- 范围：仅关注 Button 控件，不包括 ListBoxItem
- 特殊处理：播放控制栏按钮暂时使用与普通按钮相同的处理方式

## 实现计划
1. **修改 PrimaryButton 样式**（`Styles.xaml:16-41`）：
   - 移除自定义的 ControlTemplate，改为基于 Material Design 的 Button 样式
   - 添加 `materialDesign:ButtonAssist.UniformCornerRadius="6"` 保持圆角一致
   - 通过 `materialDesign:RippleAssist.FeedbackOpacity` 或样式设置按下时透明度降低效果
   - 保持现有的背景色（BrushAccent）、前景色（BrushBackground）等属性

2. **修改 SecondaryButton 样式**（`Styles.xaml:43-64`）：
   - 同样基于 Material Design Button 样式
   - 保持背景色（BrushSurface）、前景色（BrushTextPrimary）
   - 确保悬停和按下效果正确

3. **修改 IconButton 样式**（`Controls.xaml:5-30`）：
   - 评估是否保留圆形设计或使用 MD 的 Icon Button 样式
   - 如保留圆形设计，添加 MD Ripple 效果支持
   - 确保按下时透明度降低效果

4. **修改 GhostIconButton 样式**（`Controls.xaml:33-36`）：
   - 继承调整后的 IconButton 样式
   - 保持透明背景和次要前景色

5. **更新使用这些样式的按钮**：
   - `PlaybackControls.xaml` 中的播放控制按钮（PrevCommand、PlayCommand、NextCommand）
   - 其他可能使用这些样式的按钮

6. **测试与调整**：
   - 验证 Material Design 主题配置正确（`App.xaml` 中的 BundledTheme 设置）
   - 测试所有按钮的 Ripple 效果和透明度变化
   - 确保按钮在禁用状态下的表现正确
   - 检查与现有布局的兼容性

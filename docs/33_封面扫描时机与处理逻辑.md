# 封面扫描时机与处理逻辑

## 现状
扫描时已自动缓存封面，无需额外处理。

## 实现逻辑
`TrackMetadataService.FillMetadataFromFile()` 在扫描过程中会调用 `GetOrExtractCover(track)` 方法，自动完成：

1. **检查已记录路径** — 数据库中已存储且文件存在时直接返回
2. **同专辑共享** — 查找同专辑其他曲目是否已有缓存封面
3. **本地缓存** — 提取后写入缓存目录

## 缓存目录
`程序目录/Cache/Covers/`

## 缓存文件名格式
`{GUID}.jpg` — 使用 GUID 随机生成唯一文件名

## 生成逻辑
```csharp
string guidKey = Guid.NewGuid().ToString();
string cachePath = Path.Combine(cacheDir, $"{guidKey}.jpg");
```

## 调用链路
- `PlaylistManagerService.ScanPlaylist()` → `TrackMetadataService.ImportTrack()` → `FillMetadataFromFile()` → `GetOrExtractCover()`

## UI 调用（延迟提取兜底）
- 首次加载前，UI 显示时会调用 `GetOrExtractCover` 提取封面（如果扫描时未提取）
- `TrackListViewModel`、`NowPlayingViewModel`、`PlaybackControlBarViewModel` 均调用此方法

## 结论
当前实现已满足需求，但需将缓存文件名从 Base64 编码改为 GUID。

## ✅ 已完成 (Completed)

### 🚨 修复与稳定性
- [x] **崩溃诊断/解决**: 引入全局异常捕获与初始化序列优化。
- [x] **单例模式**: 已通过 `Mutex` 互斥体实现单例检测，防止多开。
- [x] **启动可靠性**: 修复 UI 刷新早于服务实例化导致的 `NullReferenceException`。
- [x] **进度控制修复**: 解决无法拖动进度条、位置无法手动更改的问题。
- [x] **切歌与停止补丁**: 修复上一首、下一首功能，修复 Stop 键重置进度条。
- [x] **歌单管理修复**: 修复无法添加新歌单、右键菜单失效问题。
- [x] **消除爆音**: 修复跳转或循环时的爆裂声。

### 🎵 引擎与性能
- [x] **反向搜索优化**: 实现了基于 SAD 的指纹匹配算法 (`FindBestLoopPoints`)。
- [x] **大规模加载优化**: 采用虚拟化与异步加载，支持 1000+ 曲目流畅显示。
- [x] **线程优化**: 音频线程调整到最高优先级。
- [x] **AB 循环支持**: 实现了 "前奏 + 循环段" 结构。

### ✨ 功能与交互
- [x] **播放模式增强**: 实现了列表循环、随机播放 (Shuffle)，以及 Play All 完整播放。
- [x] **小窗口模式**: 支持在状态栏小窗口进行基本操作和播放。
- [x] **别名系统**: 支持修改歌曲显示名称（不影响物理文件名）。
- [x] **电脑端交互优化**: 悬停显示、右键功能增强，并允许复制文字。
- [x] **列表同步**: 切歌时列表自动随之滚动至对应位置。

### 💾 数据与底层
- [x] **数据库升级**: 由 CSV 迁移至 **SQLite** (`LoopData.db`)。
- [x] **配置整合**: 整合了 `settings.conf` 与 `playlists.conf`。
- [x] **架构解耦**: `PlayerService` 层分离，MVVM 深度重构。
- [x] **资源文件体系**: 支持简中、英文，采用 `.resx` 标准方案。

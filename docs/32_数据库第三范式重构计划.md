# 更改计划

## 需求描述
CPU大人要求将数据库设计重构为符合第三范式的结构，将当前混杂所有数据的 LoopPoints 表拆分为 Tracks、Artists、Albums、LoopPoints、UserRatings 五张表。

## 实现计划
1. **设计新数据库架构**：创建 Tracks（曲目）、Artists（艺术家）、Albums（专辑）、LoopPoints（循环点）、UserRatings（用户评分）5张表
2. **更新 BaseRepository.cs**：重写表结构定义和迁移逻辑
3. **创建新数据模型**：新增 Artist、Album、UserRating 模型类
4. **更新 TrackRepository.cs**：调整 CRUD 操作以适配新架构
5. **更新 DatabaseHelper.cs**：实现数据迁移和兼容方法
6. **检查并更新相关 Services 层代码**

## 数据库架构设计（符合第三范式）

### 1. Tracks（曲目表）— 核心实体
| 字段 | 类型 | 说明 |
|------|------|------|
| Id | INTEGER PK | 主键 |
| FileName | TEXT | 文件名（唯一标识） |
| FilePath | TEXT | 文件路径 |
| DisplayName | TEXT | 显示名称 |
| TotalSamples | INTEGER | 总采样数 |
| LastModified | DATETIME | 文件修改时间 |
| CoverPath | TEXT | 封面路径 |
| AnalyzedAt | DATETIME | 分析时间，NULL表示未扫描 |
| AlbumId | INTEGER FK | 关联 Albums |

### 2. Artists（艺术家表）
| 字段 | 类型 | 说明 |
|------|------|------|
| Id | INTEGER PK | 主键 |
| Name | TEXT UNIQUE | 艺术家名（唯一） |
| CoverPath | TEXT | 艺术家封面 |

### 3. Albums（专辑表）
| 字段 | 类型 | 说明 |
|------|------|------|
| Id | INTEGER PK | 主键 |
| Name | TEXT | 专辑名 |
| ArtistId | INTEGER FK | 关联 Artists |
| CoverPath | TEXT | 专辑封面 |

### 4. LoopPoints（循环点表）
| 字段 | 类型 | 说明 |
|------|------|------|
| Id | INTEGER PK | 主键 |
| TrackId | INTEGER FK | 关联 Tracks |
| LoopStart | INTEGER | 循环起点 |
| LoopEnd | INTEGER | 循环终点 |
| LoopCandidatesJson | TEXT | 候选循环点JSON |
| AnalysisLastModified | DATETIME | 分析时间 |

### 5. UserRatings（用户评分表）
| 字段 | 类型 | 说明 |
|------|------|------|
| Id | INTEGER PK | 主键 |
| TrackId | INTEGER FK | 关联 Tracks |
| Rating | INTEGER | 评分 0-5 |
| IsLoved | INTEGER | 是否收藏 |
| LastModified | DATETIME | 更新时间 |

**关系链**：Tracks → Albums → Artists（消除传递依赖）

## 预期结果
数据库设计符合第三范式，消除数据冗余，解决传递依赖问题，提高数据一致性和查询效率。

# 更改计划

## 需求描述
CPU 大人要求优化进度条（Slider）的拖拽点（Thumb）显示效果：
1. 拖拽点外观丑陋，需要更换成主题色
2. 拖拽点只在光标靠近进度条时才显示（Hover 时显示）
3. 可考虑添加平滑的动画过渡效果

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/UI/Controls/ProgressControls.xaml` - 进度条控件界面
  - `seamless loop music/UI/Controls/ProgressControls.xaml.cs` - 进度条后台逻辑
  - `seamless loop music/UI/Themes/Colors.xaml` - 主题颜色资源（ColorAccent: #89B4FA）
  - `seamless loop music/UI/Themes/Controls.xaml` - 控件样式定义
  - `seamless loop music/UI/Themes/Styles.xaml` - 通用样式

## 当前项目状态
- 相关功能的现有代码情况：
  - ProgressControls 使用原生的 Slider 控件，未应用自定义样式
  - Slider 的 Thumb（拖拽点）使用 WPF 默认样式，与项目 Material Design 主题不协调
  - 当前 XAML 配置：
    ```xml
    <Slider x:Name="ProgressBar" Value="{Binding ProgressValue}" Maximum="1000"
            IsMoveToPointEnabled="True" Margin="0,2,0,0"
            ValueChanged="ProgressBar_ValueChanged"/>
    ```
  - 没有设置 Thumb 尺寸、颜色或可见性相关的样式

- 可复用的逻辑：
  - 项目主题色 `BrushAccent` (#89B4FA) 和 `BrushAccentHover` (#B4D0FF)
  - Colors.xaml 中已定义的颜色资源可直接使用
  - Controls.xaml 中的其他控件样式可作为视觉参考（如圆角、阴影效果）

## 可选实现方案

- 方案A：自定义 Slider Template（推荐）
  - 描述：在 Controls.xaml 中定义 `ModernSlider` 样式，重写 Slider 的 ControlTemplate，自定义 Track 和 Thumb 的外观
  - 优点：
    - 完全控制 Thumb 的外观和行为
    - 可以通过 Trigger/Storyboard 实现 Hover 时显示、离开时隐藏的动画
    - 样式可复用，其他 Slider 也能使用（如音量控制）
  - 缺点：
    - 需要编写较多的 XAML 代码
    - 需要理解 Slider 的模板结构（PART_Track, Thumb 等）

- 方案B：仅修改 Thumb 样式
  - 描述：通过 Style 单独设置 Thumb 的属性，使用 Visibility 或 Opacity 控制显示
  - 优点：
    - 实现简单，代码量少
    - 不需要重写整个 Template
  - 缺点：
    - 难以实现平滑的显示/隐藏动画
    - 对 Slider 其他部分（轨道）的样式控制有限

## 实现计划
1. 在 `UI/Themes/Controls.xaml` 中添加 `ModernSlider` 样式，包含：
   - 自定义 ControlTemplate，设置轨道高度为 3px，使用 `BrushSurfaceHighlight` 作为背景
   - 自定义 Thumb 样式：圆形、10x10 尺寸、`BrushAccent` 填充、带阴影效果
   - 设置 Thumb 的 Margin="-5,0,0,0"，使拖拽点中心与进度条末端对齐
   - 添加 DataTrigger：当鼠标悬停在 Slider 上时，Thumb 的 Opacity 从 0 变为 1（动画过渡）
   - 鼠标离开时，Thumb 的 Opacity 从 1 变回 0（完全隐藏）

2. 修改 `UI/Controls/ProgressControls.xaml`，为 Slider 应用 `ModernSlider` 样式：
   ```xml
   <Slider x:Name="ProgressBar" Style="{StaticResource ModernSlider}"
           Value="{Binding ProgressValue}" Maximum="1000"
           IsMoveToPointEnabled="True" Margin="0,2,0,0"
           ValueChanged="ProgressBar_ValueChanged"/>
   ```

3. 同步更新音量控件 `UI/Controls/VolumeControls.xaml`，为音量 Slider 也应用 `ModernSlider` 样式

4. 更新编辑界面 `UI/Views/LoopWorkspace.xaml`，为 MatchWindowSize 和 SearchRadius 两个 Slider 应用 `ModernSlider` 样式

3. 可选：同步更新音量控件的 Slider 样式（如果 VolumeControls 也使用 Slider）

## 预期结果
- 进度条轨道显示为 3px 高的细线，使用项目主题色
- 正常情况下，拖拽点不可见（Opacity=0）
- 正常情况下，拖拽点完全隐藏（Opacity=0）
- 当鼠标靠近进度条、拖拽点或正在拖拽时，拖拽点平滑显示（Opacity=1），颜色为主题色
- 鼠标离开后，拖拽点平滑隐藏（Opacity=0）
- 拖拽点为 14x14 小尺寸圆形，带有轻微阴影效果，提升视觉层次感
- 整体风格与项目的 Material Design 主题协调统一
- 音量控件的 Slider 也应用相同样式，保持 UI 一致性
- LoopWorkspace 编辑界面的两个参数调节 Slider（MatchWindowSize、SearchRadius）也应用相同样式

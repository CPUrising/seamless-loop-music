# 26_音频引擎重构与 UI 状态监听优化报告

## 1. 工作总结
本日工作的重点是解决 **Seamless Loop Music** 在 Prism/MVVM 架构迁移后的播放不稳定性及 UI 进度同步问题。通过重构底层状态通知机制，显著提升了音频播放的连贯性和 UI 响应的准确度。

## 2. 完成的核心改动

### 2.1 音频引擎 (AudioLooper) 健壮性增强
- **缓冲区优化**：针对播放乱序（碎片化）问题，将 `BufferedWaveProvider` 的缓冲区从 2s 增加至 **5s**，并将填充门槛设为 3s，有效解决了高频 I/O 下的数据丢弃问题。
- **线程安全计数**：对 `_totalBytesReadSinceSeek` 字节计数器与 `AddSamples` 动作实施了 `lock` 同步，确保进度计算在多线程环境下不再产生跳变或负值。
- **音量持久化**：修复了重新初始化播放设备时音量重置的 bug，实现了播放过程中音量的平滑继承。

### 2.2 UI 状态监听模式重构（主动转被动）
- **架构变更**：移除了底层 `AudioLooper` 昂贵的 `OnPositionChanged` 事件推送逻辑。
- **DispatcherTimer 轮询**：在 `PlaybackControlBarViewModel` 中引入 100ms 间隔的 `DispatcherTimer`，通过拉取（Pull）方式更新进度条和时间文本，减轻了音频解码线程的负担。
- **双向换算**：完善了 0-1000 比例值与实际播放时间的相互换算逻辑，支持点击进度条直接跳转。

### 2.3 导航与联动功能修复
- **LibraryView 筛选**：为 `LibraryViewModel` 实现了 `INavigationAware` 接口，使其能根据侧边栏传递的 `PlaylistId` 过滤显示曲目。
- **侧边栏精简**：去除了侧边栏冗余的歌曲列表显示，优化了歌单管理区域的视觉布局。

# 47. 文件夹导航实现报告

## 1. 需求背景
为了提供更灵活的音乐库浏览体验，我们在原有的专辑/艺术家/歌单分类基础上，新增了**文件夹导航**功能。这允许 CPU 大人直接按照硬盘上的物理目录结构来浏览并播放音乐。

## 2. 核心架构设计

### 2.1 UI 表现层 (Views)
- **LibraryView.xaml**: 
  - **根目录选择器**: 顶部的 `ComboBox` 显示所有已添加的音乐根目录，并带有 ToolTip 提示完整路径。
  - **文件夹列表**: 中间一栏显示当前目录下的子文件夹，支持点击进入。
  - **面包屑导航**: 在文件夹列表上方显示当前路径的面包屑，支持点击回退。
- **TrackListView.xaml**:
  - **路径显示**: 右侧标题栏显示当前所在的文件夹名称。
  - **曲目过滤**: 实时显示当前文件夹直属的音乐文件。

### 2.2 逻辑层 (ViewModels)
- **LibraryViewModel**:
  - 维护 `RootFolders` 和 `Subfolders` 集合。
  - 处理 `SelectedRoot` 变更逻辑：自动加载该根目录下的子文件夹，并通知右侧曲目列表更新。
  - 整合 Breadcrumbs 逻辑。
- **TrackListViewModel**:
  - 核心属性 `CurrentFolderPath`。
  - **路径过滤算法**: 采用严格的标准化路径对比：
    1. 统一斜杠方向为 `\`。
    2. 统一转为小写以忽略大小写差异。
    3. 去除末尾斜杠。
    4. 对比曲目所在目录与 `CurrentFolderPath` 是否一致。

### 2.3 服务层 (Services)
- **FoldersService**:
  - `GetRootFoldersAsync()`: 从数据库获取所有音乐根目录。
  - `GetSubfoldersAsync(path)`: 扫描数据库中该路径下的所有曲目，并提取其下一级子文件夹名称，确保只显示存在曲目的文件夹。
  - `GetBreadcrumbs(path)`: 解析路径生成面包屑对象。

## 3. 关键交互流程
1. CPU 大人点击左侧“文件夹”分类。
2. `LibraryView` 中间栏显示根目录列表。
3. 选择一个根目录后，中间栏展示子文件夹，同时右侧 `TrackListView` 刷新并显示该目录下直属的歌曲。
4. 点击子文件夹进入深层目录，面包屑实时更新。
5. 点击面包屑或“返回上级”按钮进行回退。

## 4. 已解决的技术难点
- **路径标准化**: 解决了 Windows 路径中大小写和斜杠混用的对比失败问题。
- **加载顺序**: 修复了异步加载时 Tracks 集合为空导致过滤结果不显示的竞争问题。
- **UI 布局**: 优化了长路径在侧边栏的显示效果，增加了自动修饰（Ellipsis）和 ToolTip。

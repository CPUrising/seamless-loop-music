# 更改计划

## 需求描述
在侧边栏的分类列表（专辑、艺术家）中显示小封面图片，替代目前仅显示文字图标（💿/👤）的方案，提升视觉体验。

## 需要了解的信息
- 当前项目的相关文件：
  - `seamless loop music/Models/CategoryItem.cs` — 已有 `ImagePath` 属性
  - `seamless loop music/Models/Album.cs` — 已有 `CoverPath` 属性
  - `seamless loop music/Models/Artist.cs` — 已有 `CoverPath` 属性
  - `seamless loop music/UI/ViewModels/LibraryViewModel.cs` — `LoadCategoryItems` 方法负责构建分类项
  - `seamless loop music/UI/Views/LibraryView.xaml` — 中间列 ListBox 项模板
  - `seamless loop music/Services/TrackMetadataService.cs` — 封面提取与缓存逻辑
  - `seamless loop music/Data/DatabaseHelper.cs` — 数据库含 `Albums.CoverPath` 和 `Artists.CoverPath` 字段
  - `seamless loop music/Data/Repositories/ITrackRepository.cs` — 曲目仓库接口

## 当前项目状态
- **现有实现**：
  - `CategoryItem` 模型已定义 `ImagePath` 属性（第17行），但当前 `LoadCategoryItems` 方法中（第259、271行）仅设置为 `g.FirstOrDefault()?.FilePath`（音频文件路径，非封面路径）
  - `LibraryView.xaml` 第106-108行的项模板仅显示 `Icon` 文字（如 💿、👤），未使用 `ImagePath`
  - `Albums` 表和 `Artists` 表均有 `CoverPath` 字段，但当前未正确填充和使用
  - `TrackMetadataService.GetOrExtractCover` 已能从音频文件提取封面并缓存到 `Cache/Covers/` 目录，同时会查询同专辑其他曲目的封面（第178-198行）
  - 数据库已按第三范式设计：`Tracks` → `AlbumId` → `Albums`，`Albums` → `ArtistId` → `Artists`

- **可复用的逻辑**：
  - `TrackMetadataService.GetOrExtractCover` 封面提取逻辑可直接复用
  - `Albums` 表中已有 `CoverPath` 字段，可以作为专辑封面的权威来源
  - `Artists` 表中已有 `CoverPath` 字段，可作为艺术家图片的权威来源
  - `Track.CoverPath` 属性（每首曲目独立封面）可作为回退方案

## 可选实现方案

- **方案A：在 LoadCategoryItems 中查询数据库获取封面路径（✅ CPU 大人已确认选择）**
  - 描述：在 `LibraryViewModel.LoadCategoryItems` 中，对于 Album/Artist 类型，通过 `IDatabaseHelper` 或新增仓库方法查询 `Albums`/`Artists` 表的 `CoverPath`，赋值给 `CategoryItem.ImagePath`；修改 XAML 项模板，优先显示图片，无图片时显示默认图标
  - 优点：数据来源统一（数据库），封面路径集中管理，不依赖具体曲目
  - 缺点：每次加载分类列表需额外查询数据库；若 `Albums`/`Artists` 表的 `CoverPath` 未正确填充，需先修复填充逻辑

- **方案B：使用首曲目的 CoverPath 作为分类封面**
  - 描述：在 `LoadCategoryItems` 中，取分组内第一首曲目的 `CoverPath` 作为 `CategoryItem.ImagePath`；修改 XAML 项模板显示图片
  - 优点：实现简单，无需额外数据库查询
  - 缺点：`MusicTrack` 对象当前可能未携带 `CoverPath`（需确认 `GetAllAsync` 是否包含此字段）；同专辑不同曲目的 `CoverPath` 可能不一致

- **方案C：延迟加载封面（异步）**
  - 描述：在 `CategoryItem` 中增加 `LoadImageAsync` 方法，或在 ViewModel 中异步加载封面，避免阻塞 UI
  - 优点：不会阻塞分类列表加载
  - 缺点：实现复杂，需处理异步状态和占位符

## 一、项目简介

**Seamless Loop Music Player（无缝循环音乐播放器）** 是一款专为游戏 BGM（Galgame / RPG）及环境音效（白噪音等）打造的**无缝循环播放与高效管理工具**。

核心亮点：
- 采样级精准循环对齐（自研时域互相关算法 + 集成 PyMusicLooper 引擎）
- A/B 双文件拼接播放（Intro + Loop 模式）
- 基于 `(文件名 + 总采样数)` 的音频指纹识别系统
- SQLite + Dapper 持久化后端，支持 WAL 并发
- 中英文双语 UI
- 完整的 MVVM 架构 + Repository 仓储模式
- **Prism 框架驱动**的区域化导航与依赖注入
- **Material Design 风格 UI**

支持格式：**MP3 / OGG / WAV**，支持 Windows 10 及以上。

## 四、分层架构与职责

### 4.1 Core — 音频核心层

负责底层音频的加载、解码、无缝循环播放。

| 文件 | 核心职责 |
|------|----------|
| `AudioLooper.cs` | **主控类**（partial class）。管理播放状态机（Play/Pause/Stop/Seek），持有 `LoopStream` 与 `BufferedWaveProvider`，通过 NAudio 的 `WaveOutEvent` 输出音频 |
| `AudioLooper.Loader.cs` | 文件加载。支持 WAV/OGG/MP3 单文件加载，以及 A/B 双文件在内存中物理预合并（`MemoryStream` + `RawSourceWaveStream`） |
| `AudioLooper.Analysis.cs` | 自研智能匹配算法。基于**金字塔搜索（Pyramid Search）** 的时域互相关，支持逆向（固定End找Start）与正向（固定Start找End）匹配。使用独立文件流避免与播放线程冲突 |
| `AudioLooper.Mixing.cs` | 后台填充线程。专用线程持续将 `LoopStream` 的音频数据写入 `BufferedWaveProvider`，维持 3 秒缓冲，设为 `AboveNormal` 线程优先级 |
| `LoopStream.cs` | `IWaveProvider` 实现，**无缝循环的核心**。在 `Read()` 方法中监控当前位置，到达 `LoopEndPosition` 时自动跳回 `LoopStartPosition`，通过线程锁保证线程安全 |

## 六、核心功能模块

### 6.1 无缝循环播放
- `LoopStream.Read()` 在读取音频时检测位置
- 到达 `LoopEndPosition` 自动跳回 `LoopStartPosition`
- `AudioLooper.Mixing` 后台线程持续向 `BufferedWaveProvider` 填充数据，维持 3 秒缓冲

### 6.2 智能匹配（自研）
- 基于时域互相关的金字塔搜索算法（`AudioLooper.Analysis.cs`）
- 两种模式：
  - **逆向**：以循环终点前 1 秒为指纹，在循环起点附近搜索最佳匹配
  - **正向**：以循环起点后 1 秒为指纹，在循环终点附近搜索最佳匹配
- 支持自定义匹配长度和搜索半径

### 6.3 极致匹配（PyMusicLooper）
- 通过 `PyMusicLooperWrapper` 调用外部 CLI
- 支持环境自动检测（uv / pip 安装方式）
- 返回 Top N 候选循环点，缓存到数据库的 `LoopCandidatesJson` 字段
- 支持批量操作

### 6.4 A/B 拼接模式
- 自动探测 A/B 段文件命名规则（`_A/_B`、`_a/_b`、`_intro/_loop` 等）
- 在内存中将两段音频物理合并为 `RawSourceWaveStream`
- 自动设置循环起点为 B 段开头

### 6.5 歌单与数据管理
- 两种歌单模式：**文件夹关联型**（自动扫描）和 **手动管理型**（逐个添加删除）
- 音频指纹系统：`(FileName + TotalSamples)` 双键匹配
- Repository 仓储模式提供统一的数据访问接口
- 支持离线检测、别名系统、拖拽排序持久化

### 6.6 播放模式
- 单曲循环 (SingleLoop)
- 列表循环 (ListLoop)
- 随机播放 (Shuffle)
- 可自定义循环次数后自动切歌

### 6.7 Prism 区域导航
- 基于 `Prism.Unity` 的依赖注入和区域导航
- `TrackListViewModel` 实现 `INavigationAware` 接口
- 支持按 Album / Artist / Playlist 分类过滤

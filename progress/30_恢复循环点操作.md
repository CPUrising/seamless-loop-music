# 30_恢复循环点操作

## 当前状态调查
- **背景**：在完成三栏式 UI 重构后，原本的循环点寻找功能入口变得不够直观，且存在数据同步 Bug。
- **具体问题**：
    1. 在 `TrackListView` 点击“编辑”(✎)按钮进入 `DetailView` 时，右侧的 `LoopWorkspace` 依赖 `TrackLoadedEvent` 更新。
    2. 只有在播放曲目时，`PlaybackService` 才会发布 `TrackLoadedEvent`。
    3. 如果只是进入编辑页面而不播放，`LoopWorkspace` 显示的是上一个播放曲目的信息，导致无法对当前选中的曲目进行分析。
- **功能缺失**：
    - `TrackListView` 中缺少直接触发分析的入口，必须进入详情页。

## 恢复计划
- [ ] **数据同步修复**：
    - [ ] `DetailViewModel.cs`: 在 `OnNavigatedTo` 逻辑中，增加手动触发曲目加载/选中的事件通知。
    - [ ] `LoopWorkspaceViewModel.cs`: 确保能响应从详情页传来的曲目切换信号，并及时刷新 UI。
- [ ] **按键响应修复**：
    - [ ] `LoopWorkspaceViewModel.cs`: 修复 `ExecuteAdjust` 方法中的参数分隔符（从 `|` 改为 `:`），使手动增减秒数和采样值的按键恢复响应。
- [ ] **列表入口增强**：
    - [ ] `TrackListViewModel.cs`: 增加 `AnalyzeTrackCommand`。
    - [ ] `TrackListView.xaml`: 在列表中增加“分析”按钮或上下文菜单。
- [ ] **验证循环分析流程**：
    - [ ] 确保 `PyMusicLooper` 环境检查逻辑正常。
    - [ ] 确保候选列表弹窗（`LoopListWindow`）正常显示并能应用结果。
